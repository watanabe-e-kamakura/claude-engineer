## 役割
認証・認可・外部入力処理のセキュリティリスクを専門的にチェックする。

## いつ呼ばれるか
- 認証・認可ロジックの変更があるとき
- ユーザー入力を処理する新規エンドポイントの実装時
- ファイルアップロード処理の実装時
- 外部API連携の実装時
- config.yml の stack がウェブアプリケーションのとき

## チェック観点

### 1. 認証・認可
- 認証チェックの漏れ（middleware の適用漏れ）
- 認可チェックの漏れ（他ユーザーのリソースにアクセスできないか）
- 権限昇格の可能性
- セッション管理の適切性

### 2. インジェクション
- SQL インジェクション（生クエリ、whereRaw 等）
- コマンドインジェクション（exec, shell_exec 等）
- テンプレートインジェクション
- パストラバーサル

### 3. XSS / CSRF
- ユーザー入力の出力時エスケープ
- {!! !!} や dangerouslySetInnerHTML の使用箇所
- CSRF トークンの適用
- Content-Security-Policy ヘッダー

### 4. データ保護
- 機密情報のログ出力（パスワード、トークン、個人情報）
- Mass Assignment（fillable / guarded の設定）
- エラーレスポンスでの内部情報漏洩
- 暗号化が必要なデータの平文保存

### 5. 外部連携
- API キーのハードコード
- HTTPS 強制
- レスポンスの検証（信頼できない外部データ）
- レート制限

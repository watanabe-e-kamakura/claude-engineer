# チームメンバー（分身）の作り方ガイド

## 概要

同僚の Slack メッセージ、PR レビューコメント、メール、設計書、実装コードを分析して、
`team/*.md` にメンバー定義を生成する。直近1年分程度のデータを食わせると精度が高い。

## データソースと取得方法

### 1. Slack（推奨: Slack MCP）

最もリッチなデータソース。技術議論・設計相談・障害対応のやりとりから「考え方の癖」が取れる。

**Slack MCP が利用可能な場合:**
```
〇〇さんの直近1年の #dev チャンネルでの発言を検索して、
技術議論・設計判断・レビュー依頼のメッセージを分析して。
```

Slack MCP の `slack_search_public_and_private` で対象者の発言を収集できる:
```
query: "from:@username after:2025-04-01 in:#dev-channel"
```

**Slack MCP がない場合:**
- Slack のエクスポート機能や検索結果をコピペで渡す
- 重要な議論のスレッド URL を Claude に読ませる

### 2. GitHub PR コメント（推奨: gh CLI）

レビューの癖が最も直接的に出るデータソース。何を指摘するか、どこを重視するか。

**gh CLI で取得:**
```bash
# 対象者がレビューした PR のコメントを取得（直近1年）
gh api search/issues \
  --method GET \
  -f q="is:pr reviewed-by:username org:your-org created:>2025-04-01" \
  -f per_page=50 \
  --jq '.items[].number' | while read pr; do
    gh api repos/your-org/your-repo/pulls/$pr/comments \
      --jq '.[] | select(.user.login == "username") | .body'
done
```

```
これは〇〇さんのPRレビューコメントです。
レビューの癖、よく出る指摘、判断基準を分析して team/〇〇.md を作って。
```

### 3. メール（推奨: Gmail MCP）

設計レビューのフィードバック、仕様調整のやりとりなど、Slack に残らない判断が取れる。

**Gmail MCP が利用可能な場合:**
```
〇〇さんとの直近1年のメールから、
技術的な議論・設計判断・フィードバックを抽出して。
```

Gmail MCP の `gmail_search_messages` で検索:
```
query: "from:username@example.com after:2025/04/01 subject:(設計 OR レビュー OR PR)"
```

**Gmail MCP がない場合:**
- 重要なメールの本文をコピペで渡す

### 4. 設計書・実装コード

リポジトリ内のコードや設計ドキュメントから、設計思想・実装スタイルを抽出する。

```bash
# 対象者の直近のコミットを取得
gh api search/commits \
  --method GET \
  -f q="author:username org:your-org committer-date:>2025-04-01" \
  -f per_page=30 \
  --jq '.items[].sha'
```

```
このコミット群から〇〇さんの実装スタイル（命名規則、エラーハンドリング、
抽象化の粒度、テストの書き方）を分析して team/〇〇.md に反映して。
```

## 生成プロンプト

データを収集したら、以下のプロンプトで分身を生成する:

```
以下のデータを分析して .engineer/team/〇〇.md を作成して。

フォーマット:
- 役割（1行）
- activation セクション（always: true か、file_patterns / keywords で条件指定）
- レビューの特徴（箇条書き）
- よく出る指摘（実際の発言ベース）
- 判断基準（何をどう判断するか）
- 過去に見つけた問題（実績ベースの地雷）
- 口調（どういうトーンで話すか）

データソース:
[ここに Slack / PR コメント / メール / コードを貼る]
```

## 推奨データ量

| データソース | 推奨量 | 理由 |
|------------|--------|------|
| Slack | 直近1年 | 技術議論は季節性がある。1年で一巡する |
| PR コメント | 直近50件 | レビュースタイルが安定するのに十分 |
| メール | 直近1年の技術系のみ | 量が多いのでフィルタ必須 |
| コード | 直近30コミット | 実装スタイルの傾向が見える |

## 必要なツール

| ツール | 用途 | 必須？ |
|--------|------|--------|
| `gh` CLI | PR コメント・コミット取得 | **推奨**（プラグイン内で既に使用） |
| Slack MCP | Slack メッセージ検索・取得 | 任意（なければコピペで代替可） |
| Gmail MCP | メール検索・取得 | 任意（なければコピペで代替可） |

**プラグインは MCP を同梱しない。** 既にセットアップ済みなら活用できるし、なくても gh CLI + コピペで分身は作れる。
